HOW Does Internet Work?

I have explained how does internet works in 5 principles.

Principle 1: The Need to Share Information

  1. We have information on Computer A and want it on Computer B, which is not in the same room.
  2. The simplest way is to physically carry the information (e.g., with a USB stick).
  3. This works, but it is slow and doesn't scale.
  4. We need a direct electronic connection between the computers.
  5. A cable (like Ethernet or fiber optic) can connect Computer A and B.
  6. Electrical signals or pulses of light can be sent through the cable.
  7. We agree on a code:
    • High voltage = "1"
    • Low voltage = "0"
  8. This creates the most basic network, a direct link, allowing computers to send bits (1s and 0s).

Principle 2: The Problem of Scale

  1. Works fine for two computers communicating directly.
  2. With three computers (A, B, C):
    1. Need cables from A → B, B → C, and A → C.
  3. With four computers:
    1. Need 6 cables.
  4. With 100 computers:
    1. Need 4,950 cables.
  5. Problem: Connecting every machine to every other machine directly does not scale and is physically impractical.
  6. Solution: Use a central device (Switch):
    1. Every computer connects to the switch.
    2. If Computer A wants to communicate with Computer C, it sends the message to the switch.
    3. The switch forwards the message only to Computer C.
  7. This setup creates a Local Network (LAN):
    1. Computers in close proximity (like in an office or home) can communicate efficiently.

Principle 3: The Problem of Connecting Networks

  1. Our office has a Local Network (LAN).
  2. Another office across town also has its own LAN.
  3. Problem: We cannot run a direct wire from every computer in our office to every computer in theirs—scaling fails again.
  4. Solution: Connect the networks themselves, not individual computers.
  5. Introduce a Router:
    1. A special computer connected to both networks.
    2. Its job is to pass messages (or "route" traffic) between networks.
  6. Connecting a third network:
    1. Simply connect our router to their router.
    2. This allows a large, interconnected web of networks.
  7. Fundamental principle of the Internet:
    1. The Internet is a "network of networks."
    2. It consists of millions of private and public local networks connected via routers.

Principle 4: The Need for Unique Addresses

  1. We have built a global web of interconnected networks. If I'm on my computer in my office, how do I send a message to a specific server in another country?
  2. There are millions of computers. We need a unique address for every single device.
  3. This gives rise to the need for a universal addressing system.
  4. Just like the postal service needs a unique street address for every house in the world, our network of networks needs a unique address for every connected device.
  5. This is the reason the IP Address (Internet Protocol Address) exists:
    1. It is a unique label (e.g., 142.250.184.142) assigned to each device.
    2. When a router sees a message, it looks at the destination IP address and decides which network to forward it to next.
    3. Routers don't know the full path; they only know the next best "hop" to send the message closer to its final destination.

Principle 5: The Problem of Reliability and Fairness

  1. If I want to send a large file, like a 1-hour video, it is a huge stream of data. If I try to send it all at once:
    1. I would completely monopolize the connection, and no one else could send anything until my video was done.
    2. If even a single bit of information gets corrupted during the transfer, the entire file is ruined, and I have to start over from the beginning.
  2. This is inefficient and unreliable. The logical solution is to break the large file into many thousands of small, numbered pieces called Packets.
  3. Each packet is like a tiny envelope. It contains:
    1. A small piece of the data (a "payload").
    2. The destination IP address (where it’s going).
    3. The sender's IP address (so they can reply).
    4. A number, so the receiving computer knows how to reassemble them in the correct order (e.g., "Packet 1 of 5,000").
  4. Packets can travel independently across the internet, sometimes even taking different routes.
  5. The receiving computer gathers all the packets, checks if any are missing (and requests them again if they are), and reassembles them in the correct order to recreate the original file.
  6. This system of rules for addressing, breaking down, sending, and reassembling data is called a Protocol. The main one used on the internet is TCP/IP (Transmission Control Protocol / Internet Protocol).

IP Address

IPv4 (Internet Protocol version 4)

  1. IPv4 is the classic IP address format everyone is used to seeing.
  2. Structure:
    1. It is a 32-bit number.
    2. To make it readable for humans, it is divided into four 8-bit sections, written as decimal numbers from 0 to 255.
    3. Example: 172.217.16.142 (one of Google's addresses)
  3. The Big Problem (The Why):
    1. A 32-bit number means there is a hard limit on the total number of possible addresses: 232, which is about 4.3 billion.
    2. When the internet was designed, 4.3 billion seemed like an infinite number.
    3. With the explosion of laptops, phones, servers, smart watches, and even smart refrigerators, we quickly realized we would run out.
    4. This scarcity is the single most important reason we have public and private IPs.

Public IP Address

  1. A Public IP address is your global, unique address on the internet.
  2. Analogy: The company's main, public phone number.
  3. Uniqueness: It must be globally unique. No two devices on the internet can have the same public IP address at the same time.
  4. Purpose: To be reachable from anywhere in the world. This is the address that web servers, email servers, and any other public service use.
  5. Assignment: It is assigned to you by your Internet Service Provider (ISP) (e.g., Comcast, AT&T). You don't control it; you lease it from them.
  6. Who has one? Your home router has one. The server hosting google.com has one. The server hosting Netflix has one.
  7. When you go to a site like whatismyip.com, it shows the Public IP address of your router. From the internet's perspective, your entire home network looks like a single device with that one public address.

Private IP Address

  1. A Private IP address is a local address used only within your own private network (e.g., your home Wi-Fi, your office network).
  2. Analogy: The employee's private, 4-digit extension number.
  3. Uniqueness: It only needs to be unique on your local network. For example, my laptop can have the address 192.168.1.100 on my home Wi-Fi, and your laptop can have the exact same address on your home Wi-Fi. They don't conflict because they are in separate, private networks.
  4. Purpose: To allow devices on the same local network to communicate with each other without needing a globally unique address for each one. For example, your laptop talks to your printer using private IPs, and your phone streams to your Chromecast using private IPs.
  5. Assignment: It is assigned to your devices by your own router.
  6. Reserved Ranges: The internet authorities have reserved specific ranges of addresses just for private networks:
    • 10.0.0.0 to 10.255.255.255 (used by large corporations)
    • 172.16.0.0 to 172.31.255.255 (used by medium-sized networks)
    • 192.168.0.0 to 192.168.255.255 (most common for home networks)

IPv6 (Internet Protocol version 6)

  1. IPv6 is the next generation of the Internet Protocol. Its primary purpose is to solve the address exhaustion problem of IPv4.
  2. Address Size:
    1. IPv4 uses a 32-bit address, giving us approximately 4.3 billion unique addresses.
    2. IPv6 uses a 128-bit address.
    3. The difference between 32-bit and 128-bit is exponential, not just 4x.
    4. The number of possible IPv6 addresses is 2128, which is roughly 340,000,000,000,000,000,000,000,000,000,000,000,000 (340 undecillion).
  3. IPv6 Address Format:
    1. Because it is so long, IPv6 uses hexadecimal (numbers 0-9 and letters a-f) instead of decimal numbers.
    2. An example of a full IPv6 address: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
    3. It is broken down into eight groups of four hexadecimal characters.
    4. The groups are separated by colons (:).

MAC Address (Media Access Control Address)

  1. A MAC Address is a unique, permanent serial number burned into every network-capable piece of hardware (e.g., your laptop's Wi-Fi card, your phone, smart TV, or network port on your desktop).
  2. It can be used to track devices. If someone gets your IP while you are offline, they can potentially track messages to your device.
  3. A MAC address is a 48-bit number. To make it readable for humans, it is typically written as 12 hexadecimal digits.
  4. The most common display formats are:
    1. Colon-Separated (Most Common): 3C:22:FB:A3:B4:C5
    2. Hyphen-Separated (Common on Windows): 3C-22-FB-A3-B4-C5
    3. Period-Separated (Used by Cisco and other network gear): 3c22.fba3.b4c5
    4. No Separators (Less Common): 3C22FBA3B4C5

Port Number

  1. Port numbers are numbers between 0 and 65535 that identify specific applications or services on a device.
  2. They work together with IP addresses to make sure network traffic goes to the correct service, like:
    • HTTP uses port 80
    • HTTPS uses port 443
  3. A port number is a 16-bit unsigned integer, which means:
    1. 16-bit: The number is stored using 16 binary digits (1s and 0s).
    2. Unsigned: The number cannot be negative; only positive values are allowed.
  4. Because it is 16-bit, the total number of possible ports is 216 = 65,536.

DNS (Domain Name System)

DNS is the global, distributed system that translates human-friendly domain names into computer-friendly IP addresses. Without it, you would have to memorize hundreds of IP addresses to browse the web.

Step 1: Checking the Local Cache (Do I already know this?)

  1. Browser Cache: The browser keeps a short-term memory of recently visited websites. If you visited google.com five minutes ago, the answer is probably here.
  2. Operating System (OS) Cache: If the browser doesn't have the IP, the OS (Windows, macOS, Linux) checks its own cache.
  3. Router Cache: Some routers also maintain their own DNS cache.

If the IP address is found in any of these caches, the process stops, and your browser immediately connects to the IP address. This is why websites you visit often load very quickly.

Step 2: The Recursive Resolver (Asking for Help)

  1. If the IP address is not found in any local cache, your computer asks for help from a Recursive DNS Resolver.
  2. Who is this? Usually a server run by your ISP. You can also use public resolvers like:
    • Google DNS: 8.8.8.8
    • Cloudflare DNS: 1.1.1.1
  3. What does "Recursive" mean? The server does all the work to find the IP address for you. It searches through the DNS hierarchy until it finds the answer.

Step 3: The Hierarchical Search (The Global Phonebook)

The Recursive Resolver doesn't know every IP address. It uses the global DNS system, which is like a giant hierarchy, and searches step by step from top to bottom:

  1. The Root Servers:
    • The resolver first asks one of the 13 logical Root Servers in the world.
    • It doesn’t ask for the full IP. It asks: "Where can I find the servers that manage the .com domain?"
    • The Root Server replies: "I don’t know the IP for google.com, but here is the address of the .com TLD servers."
    • The resolver then asks a Top-Level Domain (TLD) server for .com.
    • It asks: "Where are the servers that manage google.com?"
    • The TLD server replies: "I don’t know the exact IP for www.google.com, but here are the authoritative servers for google.com (e.g., ns1.google.com)."
  2. The Authoritative Name Server:
    • The resolver finally asks one of Google’s Authoritative Name Servers.
    • This server is the source of truth for google.com.
    • It replies with the final answer: "The IP address for www.google.com is 142.250.72.206."
  • The TLD Servers:

    Step 4: The Journey Back

    1. The Authoritative Name Server sends the IP address back to the Recursive Resolver.
    2. The Recursive Resolver stores (caches) this answer for a certain time, called TTL (Time-To-Live). This way, future requests for the same domain are answered quickly without repeating the full search.
    3. The Recursive Resolver sends the IP address back to your computer's operating system.
    4. Your operating system gives the IP address to your browser, so it can connect to the website.

    Step 5: The Final Connection

    1. Your browser now has the IP address of the website.
    2. It opens a direct connection to the IP address (e.g., 142.250.72.206) on the correct port (e.g., port 443 for HTTPS).
    3. The browser requests the webpage: "Please send me the webpage for www.google.com."
    4. The DNS process is now complete.
